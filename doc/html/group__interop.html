bool OnPublicCall ( AMX *  amx,
const char *  name,
cell *  params 
)

Gets called on every public function call.

This is the so called "public filter". It is called whenever the server calls amx_Exec(). The return value indicates whether corresponding callback function to the public will be executed in this plugin.

Parameters
amxThe AMX instance on which the public function is called.
nameThe name of the function.
paramsThe function's arguments as stored on the AMX stack, with the params[0] equal to the number of arguments multiplied by sizeof(cell).
Returns
If returns true the callback is executed, otherwise it's ignored.
cell sampgdk_CallNative ( AMX_NATIVE  native,
cell *  params

Calls a native function.

This function is suitable for calling simple natives that either have only value parameters or don't have any parameters at all. If you have to pass a reference or a string use sampgdk_InvokeNative() instead.

Note
The first element of params should contain the number of arguments multiplied by sizeof(cell). If the function takes no arguments, params may be NULL.
Parameters
nativeA pointer to the native function.
paramsThe params array passsed to the function.
Returns
The value returned by the function.

AMX_NATIVE sampgdk_FindNative ( const char *  name)

Finds a native function by name.

Searches for a native function with the specified name and returns its address. In order to be found the function must be registered with amx_Register() prior to the call.

This function is implemented using binary search.

Parameters
nameThe name of the native function.
Returns
The function's address or NULL if not found.

Returns all currently registered native functions.

This function can be used to retrieve the names and addresses of all native functions that were registered with amx_Register() by the server and plugins loaded prior to the call. The number parameter is filled with the total number of functions.

cell sampgdk_InvokeNative ( AMX_NATIVE  native,
const char *  format,
  ... 
)

Invokes a native function with the specified arguments.

Argument types are specified via format where each character, or specifier, corresponds to a single argument. The following format specifiers are supported:

Specifier C/C++ type Description
i int integer value
d int integer value (same as 'i')
b bool boolean value
f double floating-point value
r const cell * const reference (input-only)
R cell * non-const reference (input and output)
s const char * const string (input-only)
S char * non-const string (input and output)
Note
For the 'S' specifier, the argument passed next to it specifies the size of the string buffer. Fortunately all current SA-MP natives follow this convention.
Parameters
nativeA pointer to the native function.
formatA format string specifying the types of the arguments.
...The arguments themselves.
Returns
The value returned by the function.
